## 15.2 IEEE 802.11

### 개요
- 무선 LAN에 대한 규격을 정의
- **물리층과 데이터링크층** 포함
- **무선 이더넷**이라고 불리기도 함

### WiFi (Wireless Fidelity)
- 미국 등에서는 무선 LAN에 대한 동의어로 WiFi 사용
- WiFi는 **와이파이 연합(WiFi Alliance)** 의 인증을 받은 무선 LAN

## 네트워크 구조

### 기본 서비스 집합 (BSS, Basic Service Set)

고정/이동하는 무선국과 **접근점(AP, Access Point)** 으로 구성됨

| 종류 | 설명 |
|------|------|
| **애드 혹 네트워크 (Ad hoc Network)** | AP 없이 지국끼리 직접 연결하는 네트워크 |
| **기반구조 BSS (Infrastructure BSS)** | AP를 가진 BSS. 지국들이 AP를 통해 통신 |

```
[Ad hoc Network (BSS without AP)]     [Infrastructure BSS (BSS with AP)]
  Station   Station                       Station   Station
      \       /                               \       /
       Station                             [  AP  ]
       Station                               /     \
      /       \                          Station   Station
  Station   Station
  (AP 없이 직접 연결)                    (AP를 통해 통신)
```

---

### 확장 서비스 집합 (ESS, Extended Service Set)

- AP를 가진 **2개 이상의 BSS**로 구성됨
- **분산 시스템(Distribution System)**: BSS의 AP들을 연결하는 유선 백본

```
                    [Distribution System]
                    (유선 백본 네트워크)
                   /         |          \
              [AP]          [AP]         [AP]
             /    \        /    \       /    \
           BSS1         BSS2          BSS3
        (Station)    (Station)    (Station)
                              \
                           Server or Gateway
```

> ESS에서 지국은 BSS 간 이동(로밍) 가능

## CSMA/CA

### 개요
- **충돌 회피 반송파 감지 다중접속**
- 무선 네트워크에서 사용
- 무선은 충돌을 **감지(Detect)** 하지 못함 → **회피(Avoidance)** 방식 사용
- 숨겨진 지국 문제 해결
- 프레임간 공간, 경쟁구간, 확인응답으로 구성
- **RTS-CTS-DATA-ACK** 절차로 구성

### CSMA/CA 타이밍 다이어그램

```
                    DIFS          DIFS
                     ↓             ↓
──────────────────────────────────────────────────────────→ Time
        medium busy  │  DIFS  │contention window          │ next frame
                     │        │(randomized back-off)      │
                     ↑                                    ↑
              direct access if                      slot time
              medium is free ≥ DIFS
```

- **DIFS (DCF Interframe Space)**: 채널이 비어있어도 DIFS 시간만큼 대기 후 전송
- **경쟁 구간 (Contention Window)**: 임의의 슬롯을 선택해 추가 대기 → 충돌 확률 감소

## 숨겨진 지국 문제 (Hidden Station Problem)

### 문제 상황

```
[Range of B]    [Range of C]
     ↓               ↓
  B ──── A ──── C
  ←──────→  ←──────→
  B의 범위   C의 범위
  
B와 C는 서로 감지하지 못함
둘 다 A에게 동시에 전송 → A에서 충돌 발생
"B and C are hidden from each other with respect to A"
```

### 해결 — 핸드쉐이킹 (RTS/CTS)

```
B          A          C
│──RTS────→│          │
│          │──CTS────→│  (C에게 채널 사용 중임을 알림)
│←─────────│          │
│──Data────→│          │  (C는 CTS 받았으므로 전송 자제)
│          │──ACK─────│ (필요시)
```

- **RTS (Request To Send)**: B가 A에게 전송 요청
- **CTS (Clear To Send)**: A가 주변 모든 지국에게 브로드캐스트 → C도 수신
- C는 CTS를 받으면 일정 시간 동안 전송 자제

## NAV (Network Allocation Vector)

### 개요
- **네트워크 할당 벡터**
- 채널 사용 여부를 확인하기 전 **대기해야 하는 시간** → 충돌 회피

### CSMA/CA and NAV 동작

```
Source      Destination   All other stations
  │              │              │
  │──DIFS──→    │              │
  │──RTS────────→│              │
  │              │──SIFS──→    │
  │              │──CTS────────→│ ← NAV 설정
  │              │              │  (채널 예약 시간 알림)
  │──SIFS──→    │              │
  │──Data───────→│              │  [NAV (No carrier sensing)]
  │              │──SIFS──→    │
  │              │──ACK────────→│
                              NAV 시간 동안 채널 감지 안 함
```

> RTS/CTS 패킷에 포함된 **duration 필드**로 채널 예약 시간을 알림
> 다른 지국들은 이 시간 동안 채널 감지를 중단 → NAV

## 노출된 지국 문제 (Exposed Station Problem)

### 문제 상황

```
B       A       C       D
│       │       │       │
[Range of B]  [Range of C]
    [Range of A]  [Range of D(일부)]

A→B 전송 중일 때, C는 A의 RTS를 수신
→ C는 채널이 사용 중이라고 판단, D에게 전송 자제
→ 실제로 C→D 전송은 A→B와 겹치지 않으므로 가능한데도 불필요하게 대기
```

```
B       A       C       D
├──RTS──→       ←──RTS──┤
├──CTS──←               │
├──Data─→       ├──Data─→│  ← C can send to D
                          because this area is free
                          but C erroneously refrains
                          from sending because of received RTS
```

> 숨겨진 지국 문제와 반대 개념 — 전송해도 되는데 불필요하게 대기하는 문제

## CSMA/CA 흐름도

```
Start
  ↓
Task: Back-off
  ↓
Parameters initialized
  ↓
Wait for DIFS
  ↓
Channel free? → NO → Wait Back-off time → (다시 채널 감지)
  ↓ YES
Send RTS
  ↓

  ↓ YES

  ↓

  ↓

  ↓ YES
```

## IEEE 802.11 물리층

### OFDM (Orthogonal Frequency Division Multiplexing)

**직교주파수 분할 다중화**

- FDM과 유사하지만 **한 명의 사용자가 여러 개의 직교하는 캐리어 주파수**를 이용
- 주파수들이 서로 직교 → 간섭 최소화
- 보호대역 없이 주파수를 촘촘하게 배치 가능 → 대역폭 효율 향상
- 6장 FDM과의 차이: FDM은 각 주파수를 다른 사용자에게, OFDM은 한 사용자가 여러 주파수 동시 사용

### IEEE 802.11 표준 요약

| IEEE | Technique | Band | Modulation | Rate (Mbps) |
|------|-----------|------|------------|-------------|
| 802.11 | FHSS | 2.400-4.835 GHz | FSK | 1 and 2 |
| 802.11 | DSSS | 2.400-4.835 GHz | PSK | 1 and 2 |
| 802.11 | None | Infrared | PPM | 1 and 2 |
| 802.11a | OFDM | 5.725-5.850 GHz | PSK or QAM | 6 to 54 |
| 802.11b | DSSS | 2.400-4.835 GHz | PSK | 5.5 and 11 |
| 802.11g | OFDM | 2.400-4.835 GHz | Different | 22 and 54 |
| 802.11n | OFDM | 5.725-5.850 GHz | Different | 600 |

### 최신 Wi-Fi 표준

| 표준 | 별칭 | 속도 | 특징 |
|------|------|------|------|
| 802.11ac | Wi-Fi 5 | 433Mbps ~ 3.39Gbps | 5GHz 전용 |
| 802.11ax | Wi-Fi 6 | 1Gbps ~ 10Gbps | 2.4/5GHz |
| 802.11be | Wi-Fi 7 | 최대 46Gbps (평균 5Gbps) | **4096-QAM** |

## 블루투스 (Bluetooth)

### 개요
- 서로 다른 기능을 가진 장치를 연결하기 위해 설계된 **무선 기술**
- **IEEE 802.15** 표준안
- 무선의 **개인 영역 네트워크(PAN, Personal Area Network)** 로 정의
- **저전력**에 초점을 맞춤
- 전송 거리: **10m 이내** (증폭기 사용 시 100m)

### 블루투스 버전 발전

| 버전 | 출시 | 속도 | 특징 |
|------|------|------|------|
| 1.0 | 1998 | 1 Mbps | 최초 표준 |
| 2.0 | 2004 | 3 Mbps | - |
| 3.0 | 2009 | 24 Mbps | - |
| 4.0 | 2010 | 24 Mbps (저전력 1M) | BLE(저전력) 추가 |
| 5.0 | 2016 | 50 Mbps (저전력 2M) | 거리 확대 (40m) |
| 6.0 | 2024 | - | 위치추적 정확도 향상, 보안 강화 |

## 블루투스 네트워크 구조

### 피코넷 (Piconet)

- **최대 8개**로 구성 (1개의 **Primary**와 7개의 **Secondary**)
- Primary가 통신을 제어

```
              [Primary]
             /    |    \    \
    Secondary Secondary Secondary Secondary
        (최대 7개의 Secondary)
```

### 스캐터넷 (Scatternet)

- 여러 피코넷이 연결된 구조
- **한 장치가 두 피코넷에 동시 참여** 가능 (Primary/Secondary 역할 동시 수행)

```
[Piconet 1]              [Piconet 2]
  Primary                  Primary
  /    \                  /    \
Sec   Sec ──[Primary/]── Sec   Sec
            Secondary
            (두 피코넷에 동시 참여)
```

## 블루투스 계층 구조

```
┌─────────────────────────────────────────┐
│              Applications               │
├─────────────────────────────────────────┤
│               Profiles                  │
├──────────┬──────────────────┬───────────┤
│          │    L2CAP layer   │           │
│  Audio   │                  │  Control  │

├──────────┴──────────────────┴───────────┤
│              Radio layer                │
└─────────────────────────────────────────┘
```

> L2CAP(Logical Link Control and Adaptation Protocol): LAN에서의 LLC 부계층과 유사

## 블루투스 계층 상세

### Radio 계층
- 인터넷 모델의 **물리층**과 비슷
- 적은 전력을 사용하여 **10m 반경** 범위
- **79개의 채널**의 2.4GHz ISM 대역 사용, 채널당 1MHz씩 할당
- **주파수 도약 확산 스펙트럼 (FHSS)** 사용
  - 다른 장치나 네트워크로부터의 간섭을 피하기 위함
  - 초당 **1600번** 도약

### Baseband 계층
- LAN에서 **MAC 계층**과 유사
- **TDMA-TDD** 방식 사용

### L2CAP
- LAN에서의 **LLC 부계층**과 유사

## 블루투스 TDMA

### TDD-TDMA (Time Division Duplex)
- 블루투스는 **TDMA(Time Division Duplex TDMA)-TDD** 사용
- 종국에서 **반이중 양방향 통신** 지원 (단, 동시에 이루어지지 않음)

### 단일 종국 통신

```
시간을 625 μs의 틈새(slot)로 나눔

625 μs
├──────────────────────────────────────────→ Time
│  Hop  │  Hop  │  Hop  │  Hop  │  Hop  │
│  짝수  │  홀수  │  짝수  │  홀수  │  짝수  │
│Primary│Second.│Primary│Second.│Primary│

Primary  → 짝수 슬롯 사용
Secondary → 홀수 슬롯 사용

366 μs: 실제 데이터 전송 시간 (625 μs 중)
```

### 다중 종국 통신

- 피코넷에 하나 이상의 종국이 있는 경우
- 한 종국이 **이전 틈새의 패킷에 의해 지정**되면 그 종국이 다음 틈새에 전송

```
Primary  ─[Hop]──────────────────[Hop]──────→ Time
Secondary1 ─────[Hop]────────────────────────→ Time
Secondary2 ──────────[Hop]───────────────────→ Time
           (Primary가 지정한 Secondary가 응답)
```

## 전체 요약

### IEEE 802.11 vs 블루투스

| | IEEE 802.11 (Wi-Fi) | 블루투스 |
|---|---|---|
| 표준 | IEEE 802.11 | IEEE 802.15 |
| 네트워크 종류 | WLAN | WPAN |
| 전송 거리 | 수십~수백 m | 10m (기본) |
| 속도 | 수백 Mbps ~ 수십 Gbps | 1~50 Mbps |
| 접속 방식 | CSMA/CA | TDMA-TDD |
| 확산 방식 | OFDM, DSSS | FHSS |
| 전력 | 상대적으로 높음 | 저전력 |
| 용도 | 인터넷 접속 | 기기 간 근거리 연결 |

### 주요 문제와 해결책

| 문제 | 원인 | 해결 |
|------|------|------|
| 숨겨진 지국 문제 | B와 C가 서로 감지 못해 A에서 충돌 | RTS/CTS 핸드쉐이킹 |
| 노출된 지국 문제 | 전송 가능한데 불필요하게 대기 | (완전한 해결책 없음) |
| 충돌 회피 | 무선은 충돌 감지 불가 | CSMA/CA + NAV |
