# Data Communications — 13장. 유선 LAN

## 13.1 IEEE 표준

### IEEE standard for LANs

IEEE는 LAN 표준을 데이터링크층과 물리층으로 나눠 정의함

```
┌─────────────────────────────────────────────────────────┐
│                    Upper layers                         │
├─────────────────────────────────────────────────────────┤
│              LLC (Logical Link Control)                 │  ← 데이터링크층 위
├──────────────┬──────────────┬──────────────┬────────────┤
│  Ethernet    │  Token Ring  │  Token Bus   │    ...     │  ← MAC 서브계층
│     MAC      │     MAC      │     MAC      │            │
├──────────────┼──────────────┼──────────────┼────────────┤
│  Ethernet    │  Token Ring  │  Token Bus   │            │  ← 물리층
│  physical    │  physical    │  physical    │            │
│  layers      │  layer       │  layer       │            │
└──────────────┴──────────────┴──────────────┴────────────┘
        ↑ Transmission medium (전송 매체)
   OSI or Internet model          IEEE Standard
```

### 데이터 링크층

| 서브계층 | 역할 |
|---------|------|
| **LLC (Logical Link Control)** | 논리링크 제어, 프레임 만들기 |
| **MAC (Medium Access Control)** | 매체 접근 방식 |

### 물리층
- 사용된 물리적 매체의 구현 방법과 형태에 의존

---

### HDLC 프레임 vs LLC vs MAC 프레임

```
[HDLC frame]
┌─────────┬─────────┬───────────────┬─────┐
│ Address │ Control │ Upper-layer   │ FCS │
│         │         │ data          │     │
└─────────┴─────────┴───────────────┴─────┘

[LLC PDU (상위 데이터를 감싸는 LLC 헤더)]
┌──────┬──────┬─────────┬───────────────────┐
│ DSAP │ SSAP │ Control │ Information       │
└──────┴──────┴─────────┴───────────────────┘
DSAP: Destination service access point
SSAP: Source service access point

[MAC frame]
┌────────────┬─────────────────────────────┬─────┐
│ MAC header │ MAC payload (LLC PDU 포함)  │ FCS │
└────────────┴─────────────────────────────┴─────┘
```

> LLC는 여러 MAC 방식(이더넷, 토큰링 등)에 공통으로 사용되는 상위 인터페이스. MAC은 각 물리 매체에 맞는 접근 방식을 담당.

## 13.2 표준 이더넷

### 이더넷의 역사
- **1973년 제록스의 팔로알토 연구소(PARC)** 에서 개발

### 이더넷의 발전 4세대

```
Ethernet evolution
        │
        ├── Standard Ethernet  → 10 Mbps
        ├── Fast Ethernet      → 100 Mbps
        ├── Gigabit Ethernet   → 1 Gbps
        └── Ten-Gigabit Ethernet → 10 Gbps
```

---

### 표준 이더넷 특징
- **10Mbps**로 동작
- **CSMA/CD** 방식으로 매체에 접속
- 매체는 모든 지국들 사이에서 **서로 공유**

### MAC 부계층
- 접속 방식의 동작을 관장
- 상위 계층으로부터 수신한 데이터를 프레임으로 만들고 부호화를 위한 **PLS(Physical Layer Signaling)** 부계층으로 전달
- 접속 방법: **CSMA/CD**

---

### 802.3 MAC 프레임 구조

```
┌──────────┬─────┬─────────────┬────────────┬────────┬──────────────────┬─────┐
│ Preamble │ SFD │ Destination │   Source   │ Length │  Data and        │ CRC │
│          │     │   address   │  address   │or type │  padding         │     │
├──────────┼─────┼─────────────┼────────────┼────────┼──────────────────┼─────┤
│  7 bytes │1byte│   6 bytes   │   6 bytes  │ 2 bytes│    variable      │4bytes│
└──────────┴─────┴─────────────┴────────────┴────────┴──────────────────┴─────┘
  Physical layer header
```

| 필드 | 크기 | 설명 |
|------|------|------|
| **Preamble** | 7 bytes | 56비트의 교대 1과 0 (`10101010...`) — 수신측 동기화용 |
| **SFD** | 1 byte | Start Frame Delimiter, 프레임 시작 구분자 (`10101011`) |
| **Destination address** | 6 bytes | 목적지 MAC 주소 |
| **Source address** | 6 bytes | 발신지 MAC 주소 (항상 유니캐스트) |
| **Length or type** | 2 bytes | 페이로드 길이 또는 상위 프로토콜 타입 |
| **Data and padding** | 가변 | 실제 데이터. 최소 46바이트 미만이면 패딩 추가 |
| **CRC** | 4 bytes | 오류 검출용 (CRC-32) |

> - 프레임은 **7개 필드**로 구성
> - **확인응답을 제공하지 않으므로 신뢰성이 없음** → 확인응답은 상위계층에서 구현

---

### 프레임 길이 제한

```
┌─────────────┬────────────┬────────┬──────────────────────┬─────┐
│ Destination │   Source   │ Length │  Data and padding    │ CRC │
│  address    │  address   │  PDU   │                      │     │
│   6 bytes   │   6 bytes  │ 2 bytes│   46 ~ 1500 bytes    │4bytes│
└─────────────┴────────────┴────────┴──────────────────────┴─────┘
  Minimum payload length: 46 bytes
  Maximum payload length: 1500 bytes

  Minimum frame length: 512 bits = 64 bytes
  Maximum frame length: 12,144 bits = 1518 bytes
```

| | 값 |
|---|---|
| 최소 페이로드 | 46 bytes |
| 최대 페이로드 | 1500 bytes |
| **최소 프레임 길이** | **64 bytes (512 bits)** |
| 최대 프레임 길이 | 1518 bytes |

> - **최소값 제한 이유**: CSMA/CD의 정확한 동작을 위함
>   - 충돌 감지를 위해 프레임 전송 시간 ≥ 2 × 전파 지연 시간이어야 함
>   - 12장 Example 12.5에서 계산한 64bytes와 동일

---

### 이더넷 주소 (MAC 주소)

#### Example — 16진수 표기
```
06 : 01 : 02 : 01 : 2C : 4B

6 bytes = 12 hex digits = 48 bits
```

- 48비트 주소
- 16진수로 표기 (콜론으로 구분)
- 앞 3바이트: 제조사 ID / 뒤 3바이트: 장치 고유번호

#### 유니캐스트, 멀티캐스트, 브로드캐스트

| 종류 | 설명 | 첫 바이트 LSB |
|------|------|--------------|
| **유니캐스트** | 특정 한 장치에게 전송 | 0 |
| **멀티캐스트** | 특정 그룹에게 전송 | 1 |
| **브로드캐스트** | 모든 장치에게 전송 | `FF:FF:FF:FF:FF:FF` |

> - **발신지 주소는 항상 유니캐스트**
> - 목적지 주소는 유니캐스트, 멀티캐스트, 브로드캐스트 모두 가능

```
주소 구분:
Byte 1   Byte 2   ...   Byte 6
  │
  └── LSB = 0 → 유니캐스트
      LSB = 1 → 멀티캐스트
```

---

### 표준 이더넷 구현 방식

```
Standard Ethernet common implementations
        │
        ├── 10Base5  (Bus, thick coaxial)
        ├── 10Base2  (Bus, thin coaxial)
        ├── 10Base-T (Star, UTP)
        └── 10Base-F (Star, fiber)
```

| Characteristics | 10Base5 | 10Base2 | 10Base-T | 10Base-F |
|----------------|---------|---------|----------|----------|
| Media | Thick coaxial | Thin coaxial cable | 2 UTP | 2 Fiber |
| Maximum length | 500 m | 185 m | 100 m | 2000 m |
| Line encoding | Manchester | Manchester | Manchester | Manchester |

> **명칭 해석**: `10Base-T` → 10(Mbps) Base(기저대역) T(Twisted pair)

---

### 표준 이더넷 부호화

```
[Station]
  10 Mbps data
       ↓
  Manchester encoder
       ↓
  Twisted pairs or fibers
       ↓
  Manchester decoder
       ↓
  10 Mbps data
  [Station]
```

- 표준 이더넷은 **맨체스터 부호화** 사용
- 비트 중간에 반드시 전이 발생 → 동기화 및 직류 성분 제거

### 10Base-T 구현

```
10Base-T ─── Twisted pair ─── Hub ─── Twisted pair ─── 10Base-T
                                │
                           10Base-T hub
                         ┌──────────────┐
                         │ [PC][PC][PC] │
                         └──────────────┘
                         Two pairs of UTP cable
```
- **스타형 토폴로지** 사용
- 허브(Hub)를 중심으로 UTP 케이블로 연결
- Baseband (digital)

## 13.3 표준 변경

### 대역폭 공유 문제

초기 이더넷은 모든 지국이 동일 채널 공유 → 지국 수 증가 시 성능 저하

```
[브리지 없는 경우]
  10 Mbps 전체를 모든 지국이 나눠 씀
  → 지국이 많아질수록 각 지국 실질 대역폭 감소

[브리지 있는 경우]
  10 Mbps │ 10 Mbps
  지국1   │   지국2
  세그먼트│  세그먼트
  ──────[Bridge]──────
  → 세그먼트 간 충돌 영역 분리
  → 각 세그먼트에서 독립적으로 10 Mbps 사용
```

---

### 브리지형 이더넷 (Bridged Ethernet)

- 이더넷 발전의 **첫 번째 단계**
- 브리지들은 **대역폭의 증가**와 **충돌 영역의 분리** 역할

#### Figure 12.17 — A network with and without a bridge
```
[Without bridging]
PC─PC─PC─PC─PC─PC─PC─PC  (모든 지국이 동일 충돌 영역)

[With bridging]
PC─PC─PC─PC ─[Bridge]─ PC─PC─PC─PC
  충돌 영역1               충돌 영역2
  → 세그먼트 분리로 충돌 감소, 대역폭 증가
```

---

### 스위치형 이더넷 (Switched Ethernet)

```
              [Switch]
             /    |    \
        Domain  Domain  Domain
          │       │       │
        지국들  지국들  지국들
```

- 스위치가 각 포트를 독립적인 충돌 영역으로 분리
- 동시에 여러 포트 간 통신 가능
- 각 도메인(포트)이 **전용 대역폭** 사용

---

### 전이중 스위치형 이더넷 (Full-duplex Switched Ethernet)

```
        Transmit ──→            ←── Transmit
  PC ══════════════ [Switch] ══════════════ PC
        Receive  ←──            ──→ Receive
```

- 송신과 수신이 **동시에** 가능 (전이중)
- 충돌이 발생하지 않으므로 CSMA/CD 불필요
- 각 방향으로 전체 대역폭 사용 가능

---

## 13.4 고속 이더넷 (Fast Ethernet)

### 표준: IEEE 802.3u

### 고속 이더넷의 특징
- 데이터율 **100Mbps** 향상
- 표준 이더넷과의 **호환성** 구축
- 동일한 **48비트 주소체계** 유지
- 동일한 **프레임 형식** 유지
- 동일한 **최소/최대 프레임 길이** 유지
- **자동협상 기능** — 10M와 100M 장비가 통신 가능

---

### 고속 이더넷 구현 방식

```
Common Fast Ethernet implementations
        │
        ├── 100Base-TX  (Two wires, category 5 UTP or STP)
        ├── 100Base-FX  (Two wires, fiber)
        └── 100Base-T4  (Four wires, category 3 UTP)
```

| Characteristics | 100Base-TX | 100Base-FX | 100Base-T4 |
|----------------|------------|------------|------------|
| Media | Cat 5 UTP or STP | Fiber | Cat 4 UTP |
| Number of wires | 2 | 2 | 4 |
| Maximum length | 100 m | 100 m | 100 m |
| Block encoding | 4B/5B | 4B/5B | 8B/6T |
| Line encoding | MLT-3 | NRZ-I | NRZ-4 |

> **4B/5B 블록 부호화**: 4비트 데이터를 5비트로 변환 → 직류 성분 제거, 동기화 보장
> 맨체스터 대신 4B/5B + MLT-3 방식 사용 → 대역폭 효율 향상

---

## 13.5 기가비트 이더넷 (Gigabit Ethernet)

### 표준: IEEE 802.3z

### 기가비트 이더넷의 특징
- 데이터 전송률 **1Gbps** 향상
- 표준 또는 고속이더넷과 **호환성** 구축
- 동일한 **48비트 주소** 사용
- 동일한 **프레임 형식** 사용
- 동일한 **최소와 최대 프레임 길이** 유지
- 고속 이더넷에서 정의된 것과 같은 **자동협상** 지원

---

### 전이중 / 반이중 모드

| 모드 | 상태 |
|------|------|
| **전이중 모드** | 주로 사용 |
| **반이중 모드** | 거의 사용 안 함 |

---

### 반이중 모드에서의 문제 및 해결

최소 프레임 유지 시 전송 거리 **25m** 로 매우 짧아짐 → 두 가지 방법으로 해결

#### ① 반송파 확장 (Carrier Extension)
- 최소 프레임을 **512바이트로 확장**
- 짧은 프레임에 패딩을 붙여 전송 시간을 늘림
- 전송 거리 확대 가능

#### ② 프레임 버스팅 (Frame Bursting)
- **다수의 프레임을 연속으로 전송**
- 짧은 프레임 여러 개를 한 번에 묶어서 전송
- 채널 효율 향상

---

### 기가비트 이더넷 구현 방식

```
Gigabit Ethernet implementations
        │
        ├── 1000Base-SX  (Two-wire short-wave fiber)
        ├── 1000Base-LX  (Two-wire long-wave fiber)
        ├── 1000Base-CX  (Two-wire copper STP)
        └── 1000Base-T   (Four-wire UTP)
```

| Characteristics | 1000Base-SX | 1000Base-LX | 1000Base-CX | 1000Base-T |
|----------------|-------------|-------------|-------------|------------|
| Media | Fiber short-wave | Fiber long-wave | STP | Cat 5 UTP |
| Number of wires | 2 | 2 | 2 | 4 |
| Maximum length | 550 m | 5000 m | 25 m | 100 m |
| Block encoding | 8B/10B | 8B/10B | 8B/10B | - |
| Line encoding | NRZ | NRZ | NRZ | 4D-PAM5 |

> **4D-PAM5**: 4개 회선 동시 전송, 5준위 신호 → 1Gbps 달성 (4장에서 배운 내용)
> **8B/10B**: 8비트를 10비트로 변환 → 직류 성분 제거, 동기화 보장

---

## 전체 요약

| 세대 | 표준 | 속도 | 접속방식 | 부호화 |
|------|------|------|---------|--------|
| 표준 이더넷 | IEEE 802.3 | 10 Mbps | CSMA/CD | 맨체스터 |
| 고속 이더넷 | IEEE 802.3u | 100 Mbps | CSMA/CD | 4B/5B + MLT-3 |
| 기가비트 이더넷 | IEEE 802.3z | 1 Gbps | 전이중(주) | 8B/10B, 4D-PAM5 |

### 이더넷 발전 방향
```
충돌 영역 공유 (표준)
    ↓
브리지로 세그먼트 분리
    ↓
스위치로 포트별 독립 (스위치형)
    ↓
전이중으로 CSMA/CD 불필요 (Full-duplex)
    ↓
속도 향상 (100M → 1G → 10G)
```
