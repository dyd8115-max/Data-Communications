## 2.2 OSI 모델

모든 유형의 컴퓨터 시스템 간 통신을 허용하는 네트워크 설계를 위한 **7계층 구조**

### OSI 7계층

| 계층 | 이름 | 핵심 기능 | 전달 단위 |
|---|---|---|---|
| 7 | 응용층 (Application) | 사용자 인터페이스, 네트워크 접근 | - |
| 6 | 표현층 (Presentation) | 암호화/복호화, 데이터 압축, 구문·의미 변환 | - |
| 5 | 세션층 (Session) | 세션 관리, 동기화, 대화 제어 | - |
| 4 | 전송층 (Transport) | 프로세스 대 프로세스 전달, 포트 주소, 분할/재조립 | 세그먼트 |
| 3 | 네트워크층 (Network) | 발신지-목적지 전달, 논리 주소, 라우팅 | 패킷 |
| 2 | 데이터 링크층 (Data Link) | 노드 대 노드 전달, 물리 주소, 흐름·오류 제어 | 프레임 |
| 1 | 물리층 (Physical) | 비트 스트림 전송, 전기·기계적 특성 | 비트 |

### 계층별 상세

**물리층 (Physical Layer)**
- 비트 스트림을 전자기 신호로 변환하여 매체를 통해 전송
- 기계적·전기적 특성 담당 (케이블, 커넥터)

> **비트 스트림**: 비트(0과 1)가 연속으로 나열된 데이터 흐름

**데이터 링크층 (Data Link Layer)**
- 노드 대 노드(hop-to-hop) 전달 책임
- 프레임 구성, 물리 주소 지정, 흐름·오류·접근 제어

**네트워크층 (Network Layer)**
- 발신지-목적지 패킷 전달 책임 (데이터 링크층은 노드 간)
- 논리 주소 지정 (IP), 라우팅

**전송층 (Transport Layer)**
- 전체 메시지의 프로세스 대 프로세스 전달
- 포트 주소 지정, 분할/재조립, 연결 제어, 흐름·오류 제어

**세션층 (Session Layer)**
- 세션 관리, 동기화, 대화 제어, 원활한 종료

**표현층 (Presentation Layer)**
- 두 시스템 간 데이터 구문(syntax)·의미(semantic) 처리
- 암호화/복호화, 데이터 압축

**응용층 (Application Layer)**
- 사용자가 네트워크에 접근할 수 있도록 인터페이스 제공
- 원격 로그인, 파일 전송, 메일, WWW

### OSI 모델의 실패 이유
- TCP/IP가 이미 완전히 자리잡은 후 OSI 완성
- 일부 계층이 완전히 정의되지 않음
- 전환 필요성을 충분히 보여주지 못함

> 현실에서는 OSI 모델은 개념/교육용으로만 쓰이고, 실제 네트워크는 TCP/IP 프로토콜을 사용함

## 2.3 TCP/IP 프로토콜

OSI 7계층에서 **세션층과 표현층 없음** → 5계층 구조

| TCP/IP 계층 | OSI 대응 |
|---|---|
| 응용층 | 응용 + 표현 + 세션 |
| 전송층 | 전송 |
| 인터넷층 | 네트워크 |
| 링크층 | 데이터 링크 + 물리 |

### 주소 체계

| 계층 | 주소 종류 | 설명 |
|---|---|---|
| 응용층 | 이름 (Name) | 도메인 등 사람이 읽는 주소 |
| 전송층 | 포트 주소 (Port) | 특정 프로세스 식별 |
| 인터넷층 | 논리 주소 (IP) | 발신지-목적지 식별 |
| 링크층 | 물리 주소 (MAC) | 노드 간 홉 식별 |

### Switch vs. Router

| | Switch | Router |
|---|---|---|
| 계층 | 물리/링크 (2계층) | 물리/링크/네트워크 (3계층) |
| 주소 | 물리 주소 (MAC) | 논리 주소 (IP) |
| 규모 | 소규모, 저가 | 중대형, 고가 |
| 용도 | 사용자 연결 | 외부망 연결 |
