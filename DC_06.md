# Data Communications — 5. 아날로그 전송 (변복조)

## 5.1 디지털-대-아날로그 전환

디지털 신호를 아날로그 신호로 변환하는 과정

---

## 5.2 반송파 신호 3가지 특성

반송파(Carrier Signal): 아날로그 전송에서 정보 신호의 기반이 되는 고주파 신호

| 특성 | 설명 | 변조 방식 |
|---|---|---|
| **진폭 (Amplitude)** | 파형의 최대 높이 (신호 세기) | ASK |
| **주파수 (Frequency)** | 1초당 파형 반복 횟수 (Hz) · 높을수록 촘촘 | FSK |
| **위상 (Phase)** | 파형이 시작하는 위치 (°) · 같은 파형도 출발점이 다름 | PSK |

> - **위상 예시**: φ=0° → 0에서 시작 / φ=180° → 반대 방향에서 시작
> - 비유: 같은 트랙을 같은 속도로 달려도 출발 위치가 다르면 위치가 계속 어긋남

---

## 5.3 변환 유형 & 기본 개념

| 방식 | Full Name | 변조 요소 |
|---|---|---|
| ASK | Amplitude Shift Keying | 진폭 변화 |
| FSK | Frequency Shift Keying | 주파수 변화 |
| PSK | Phase Shift Keying | 위상 변화 |
| QAM | Quadrature Amplitude Modulation | 진폭 + 위상 동시 변화 |

$$S = \frac{N}{r}$$

| 용어 | 설명 | 단위 |
|---|---|---|
| 비트율 (N) | 초당 전송되는 비트 수 | bps |
| 신호율 (S) | 초당 신호 수 | Baud |
| r | 신호 요소당 비트 수 | - |
| L | 신호 준위 수 (L = 2^r) | - |

> **예시 5.1)** r=4, S=1000 → N = 1000 × 4 = **4,000 bps**
> **예시 5.2)** N=8000, S=1000 → r=8, L=2^8=**256**

## 5.4 변조 방식 상세

### ASK (진폭편이 변조)
- 진폭만 변화 · 주파수·위상 유지
- OOK: 0 → 진폭 없음, 1 → 진폭 A

### FSK (주파수편이 변조)
- 주파수만 변화 · 진폭·위상 유지
- BFSK: 0 → 저주파(f₁), 1 → 고주파(f₂)

### PSK (위상편이 변조)
- 위상만 변화 · 진폭·주파수 유지
- BPSK: 0 → 0°, 1 → 180°
- QPSK: 4위상 → 신호당 **2비트** (45°, 135°, 225°, 315°)

### QAM (구상진폭 변조)
- ASK + PSK 조합 · 진폭·위상 동시 변화
- 신호당 더 많은 비트 → 고속 통신에 유리 (케이블 모뎀, DSL)

---

## 5.5 세부 변조 방식 정리

### OOK (On-Off Keying) — ASK의 특수 형태
- 1 = 신호 있음 (진폭 A) / 0 = 신호 없음 (진폭 0)
- 구현 쉽지만 잡음에 취약 (0과 잡음 구분 어려움)
- 사용: 광통신, 적외선 리모컨

> - **ASK와 차이**: 일반 ASK는 0일 때 진폭이 작은 값 · OOK는 진폭이 완전히 0 → ASK의 극단적 형태

### BFSK (Binary FSK)
- 주파수 2개 사용: 0 → f₁(저주파), 1 → f₂(고주파)
- ASK보다 잡음에 강함 (진폭이 아닌 주파수로 구분)
- 사용: 모뎀, 무선 통신

> - **FSK와 차이**: FSK는 여러 주파수 사용 가능 · BFSK는 딱 2개(Binary)만 사용 → FSK의 가장 단순한 형태

### BPSK (Binary PSK)
- 위상 2개 사용: 0 → 0°, 1 → 180° (정반대 방향)
- 신호당 1비트 · 구현 단순 · 잡음에 강함
- 성운도: X축 양쪽 2개 점

> - **PSK와 차이**: PSK는 위상 개수 제한 없음 · BPSK는 딱 2개(Binary)만 사용 → PSK 중 가장 단순하고 잡음에 강함 · 효율은 낮음

### QPSK (Quadrature PSK)
- 위상 4개 사용: 45°, 135°, 225°, 315°
- 신호당 2비트 → BPSK의 2배 효율
- 성운도: 4개 점 (대각선 배치)
- 예) 45°=00, 135°=01, 225°=10, 315°=11

> - **BPSK와 차이**: BPSK는 위상 2개 → 1bit/신호 · QPSK는 위상 4개 → 2bit/신호 → 같은 대역폭으로 2배 데이터 전송 가능

### 16-QAM / 64-QAM
- 진폭 × 위상 조합으로 점 수 늘림
- 16-QAM: 4×4 = 16가지 → 신호당 4비트
- 64-QAM: 8×8 = 64가지 → 신호당 6비트
- 점↑ → 효율↑ → 점 간격 좁아져 잡음에 취약↑

> - **QAM과 차이**: QAM은 진폭+위상 조합 방식의 총칭 · 앞 숫자(16, 64)는 조합 개수 → 숫자↑ = 효율↑ = 잡음 취약↑

### 구상진폭 변조 (QAM) 심화
- ASK(진폭)와 PSK(위상)를 **동시에** 변화시켜 더 많은 비트를 한 신호에 담음
- 같은 대역폭으로 최대한 많은 데이터를 보내는 게 목표

| 방식 | 진폭 종류 | 위상 종류 | 조합 | 비트/신호 |
|---|---|---|---|---|
| 4-QAM (=QPSK) | 1 | 4 | 4 | 2 |
| 16-QAM | 4 | 4 | 16 | 4 |
| 64-QAM | 8 | 8 | 64 | 6 |
| 256-QAM | 16 | 16 | 256 | 8 |

> - 현실에서 LTE는 64-QAM, LTE-A/5G는 256-QAM까지 사용
> - 잡음이 적은 환경일수록 더 높은 QAM 사용 가능 → 신호 품질에 따라 동적으로 변경
> - 비유: 같은 트럭(대역폭)에 짐을 더 촘촘하게 쌓는 것 → 많이 실을수록 흔들리면 쏟아짐

## 5.5 성운 그림 (Constellation Diagram)

신호의 진폭과 위상을 2차원 좌표로 표현

- X축: 동위상 반송파 · Y축: 구상 반송파
- 점 위치 → 진폭(원점 거리) + 위상(각도) 동시 표현

| 방식 | 점 수 | 비트/신호 |
|---|---|---|
| BPSK | 2 | 1 |
| QPSK | 4 | 2 |
| 16-QAM | 16 | 4 |
| 64-QAM | 64 | 6 |

> 점↑ → 효율↑ → 잡음에 취약↑ (트레이드오프)
