# Data Communications — 6. 다중화와 확산 대역

## 6.1 다중화 (Multiplexing)

단일 데이터 링크를 통해 여러 개의 신호를 동시에 전송하기 위한 기술

> - **왜 필요한가**: 전송 매체의 대역폭이 개별 신호보다 훨씬 크기 때문에 여러 신호를 동시에 실어 효율을 높임
> - 비유: 고속도로(링크)에 차선(채널)을 여러 개 만들어 여러 차(신호)가 동시에 달리게 하는 것

---

### 핵심 용어

| 용어 | 설명 |
|---|---|
| **다중화기 (MUX)** | - 여러 전송 스트림을 단일 스트림으로 결합 (many to one) |
| **다중복구기 (DEMUX)** | - 단일 스트림을 각각의 요소로 분리 (one to many)<br>- 전송 스트림을 해당 수신장치에 전달|
| **링크 (Link)** | - 물리적인 경로 |
| **채널 (Channel)** | - 한 쌍의 장치 간 전송을 위한 하나의 경로 |

```
[장치1] ─┐
[장치2] ──┤→ [MUX] ──────링크────── [DEMUX] ┤→ [장치1]
[장치3] ─┘                                   ├→ [장치2]
                                              └→ [장치3]
```

---

### 다중화의 범주

| 방식 | 종류 | 설명 |
|---|---|---|
| **FDM** | 주파수 분할 다중화 | 아날로그 · 주파수 대역으로 분할 |
| **TDM** | 시분할 다중화 | 디지털 · 시간 슬롯으로 분할 |
| **확산 대역** | FHSS / DSSS | 무선 통신용 · 여분 정보 추가 |

---

## 6.2 주파수 분할 다중화 (FDM)

**FDM:**

- 링크의 대역폭이 전송되는 조합 신호의 대역폭보다 클 때 적용하는 **아날로그 기술**
- 각 신호를 서로 다른 주파수 대역에 실어 동시에 전송
- 신호가 겹치지 않도록 **보호대역(Guard Band)** 을 두어야 함

> - **보호대역**: 채널 간 간섭을 막기 위한 빈 주파수 구간(ex: 차선 경계선)
> - 보호대역이 넓을수록 간섭 줄어들지만 대역폭 낭비 증가

---

### FDM 처리 과정

```
1. 각 장치에서 신호 발생 (비슷한 주파수 대역)
        ↓
2. 각 신호를 서로 다른 반송 주파수로 변조 (f1, f2, f3)
        ↓
3. 변조된 신호들을 하나의 링크로 합성 (MUX)
        ↓
4. 각 신호를 분리하여 수신기에 전달 (DEMUX)
        ↓
5. 복조하여 원래 신호 복원
```

---

### FDM 응용 사례

| 서비스 | 주파수 대역 | 채널당 대역폭 |
|---|---|---|
| AM 라디오 | 535kHz ~ 1,605kHz | 10kHz |
| FM 라디오 | 87.5MHz ~ 108MHz | 200kHz |
| TV (HD) | 470MHz ~ 806MHz | 6MHz |
| TV (UHD) | 700MHz | 30MHz |
| LTE | 800MHz ~ 2.6GHz | 20MHz |
| 5G | 3.5GHz | 100MHz |

---

## 6.3 시분할 다중화 (TDM)

**TDM (Time-Division Multiplexing)**

- 송수신 장치에 요구되는 데이터 전송률보다 전송 매체의 데이터 전송률이 클 때 적용하는 **디지털 기술**
- 시간을 여러 슬롯으로 나눠 각 채널에 순서대로 할당

> - FDM vs TDM:
>   - FDM = 주파수 대역을 나눔 (아날로그)
>   - TDM = 시간을 나눔 (디지털)

---

### 핵심 용어

| 용어 | 설명 |
|---|---|
| **시간 슬롯 (Time Slot)** | 각 채널에 할당된 시간 구간 |
| **프레임 (Frame)** | 각 채널의 슬롯을 모은 한 묶음 |
| **끼워넣기 (Interleaving)** | 스위치가 장치들을 일정 비율로 정해진 순서대로 순환 |

```
프레임 구조:
┌────┬────┬────┬────┐
│ CH1│ CH2│ CH3│ CH4│  ← 1 프레임
└────┴────┴────┴────┘
```

---

### 예시 (Example 6.8)

4개 채널을 TDM으로 다중화, 각 채널 전송률 100 bytes/s, 채널당 1바이트씩 전송

| 항목 | 계산 | 결과 |
|---|---|---|
| 프레임 크기 | 4채널 × 1byte | 4 bytes = 32 bits |
| 프레임 전송률 | 채널당 100 bytes/s | 100 frames/s |
| 링크 비트율 | 100 × 32 | **3,200 bps** |

---

### 빈 슬롯 (Empty Slots)

- 발신자가 전송할 데이터가 없으면 해당 슬롯이 비어있게 됨
- 일반 TDM의 단점 → **통계적 TDM**으로 해결

---

### 프레임 동기화 (Frame Synchronization)

- 수신측이 프레임 경계를 정확히 파악해야 함
- **프레임 구성비트 (Framing Bits)**: 각 프레임 앞에 동기화 비트 추가
- 송수신측이 같은 타이밍을 유지하도록 함

---

### DS (Digital Signal) 서비스 — T회선

북미 표준 디지털 신호 계층 구조

| 회선 | 데이터 전송률 | 음성 채널 수 |
|---|---|---|
| DS-0 | 64 Kbps | 1 |
| DS-1 (T-1) | 1.544 Mbps | 24 |
| DS-2 (T-2) | 6.312 Mbps | 96 |
| DS-3 (T-3) | 44.736 Mbps | 672 |
| DS-4 (T-4) | 274.176 Mbps | 4,032 |

> - **T-1 프레임 구조**: 24채널 × 8비트 + 1 프레임 비트 = 193비트/프레임
> - 프레임 전송률: 8,000 frames/s → 193 × 8,000 = **1.544 Mbps**

---

### E 회선 (유럽/국내 표준)

| 회선 | 데이터 전송률 | 음성 채널 수 |
|---|---|---|
| E-1 | 2.048 Mbps | 30 |
| E-2 | 8.448 Mbps | 120 |
| E-3 | 34.368 Mbps | 480 |
| E-4 | 139.264 Mbps | 1,920 |

> - 국내는 T회선이 아닌 **E회선** 사용
> - E-1이 T-1보다 더 많은 채널 지원 (30 vs 24)

---

### 통계적 TDM (Statistical TDM)

일반 TDM과 비교:

| | 일반 TDM | 통계적 TDM |
|---|---|---|
| 슬롯 할당 | 고정 (데이터 없어도 할당) | 동적 (데이터 있을 때만 할당) |
| 빈 슬롯 | 발생 | 없음 |
| 효율 | 낮음 | 높음 |
| 주소 정보 | 불필요 | 필요 (슬롯에 주소 포함) |

> - 통계적 TDM은 데이터가 있는 채널에만 슬롯 할당 → 대역폭 낭비 없음
> - 단, 각 슬롯에 어느 채널인지 주소 정보가 필요

---

## 6.4 확산 대역 방식 (Spread Spectrum)

- 무선 통신 응용을 위해 설계
- 신호에 **여분의 정보를 추가**하여 대역폭을 의도적으로 확산
- 목적: 도청 방지, 간섭 저항, 보안 강화

> - 역설적으로 대역폭을 더 많이 쓰는데 오히려 더 안전하고 간섭에 강해짐
> - 비유: 여러 주파수로 분산해서 보내면 일부가 방해받아도 전체 신호 복원 가능

---

### FHSS (주파수 뛰기 대역 확산)

**Frequency Hopping Spread Spectrum**

- 송신측이 M개의 신호를 **서로 다른 반송파 주파수를 사용**하여 변조
- 주파수가 **의사 난수 순서 (Pseudo-random)** 에 따라 계속 바뀜
- 송수신측이 같은 주파수 뛰기 순서를 공유해야 함

```
시간:  t1    t2    t3    t4    t5
주파수: f3 → f1 → f4 → f2 → f3 → ...
```

> - **장점**: 특정 주파수를 도청하거나 방해해도 전체 신호에 영향 적음
> - **사용**: 블루투스, 군사 통신

---

### FHSS 사이클

- 전체 주파수 대역을 M개로 나눔
- 각 호핑 주기(Th)마다 주파수 변경
- 한 사이클 = 모든 M개 주파수를 한 번씩 사용

---

### FHSS 대역폭 공유

- 서로 다른 사용자가 다른 주파수 뛰기 순서를 사용하면 같은 대역폭을 공유 가능
- 충돌이 가끔 발생하지만 전체적으로 복원 가능

---

### DSSS (직접 순열 확산 방식)

**Direct Sequence Spread Spectrum**

- 각 데이터 비트를 **확산 코드(Spreading Code)** 를 사용하여 n개의 비트로 대체
- 원래 1비트 → n비트 (칩, Chip)로 확산

```
원래 데이터:  1        0
확산 코드:    11011    00100  (예시)
전송 신호:    11011    00100
```

> - **칩 (Chip)**: 확산 후 생성된 비트
> - **칩 전송률**: 원래 비트율 × n배
> - **장점**: 잡음/간섭에 매우 강함, 도청 어려움
> - **사용**: CDMA, GPS, Wi-Fi (802.11b)

---

### FHSS vs DSSS 비교

| | FHSS | DSSS |
|---|---|---|
| 방식 | 주파수를 계속 바꿈 | 비트를 확산 코드로 대체 |
| 복잡도 | 낮음 | 높음 |
| 잡음 저항 | 보통 | 강함 |
| 사용 | 블루투스 | CDMA, GPS, Wi-Fi |

---
