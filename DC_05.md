# Data Communications — 4. 디지털 전송

## 4.1 회선 부호화 (Line Coding)

2진 비트 데이터를 디지털 신호로 변환하는 작업

---

### 기본 개념

| 용어 | 설명 | 단위 |
|---|---|---|
| 데이터 요소 | 데이터를 나타내는 가장 작은 단위 | 비트 (bit) |
| 신호 요소 | 디지털 신호의 가장 짧은 단위 | - |
| 데이터 전송률 (N) | 1초당 전송된 비트 수 | bps |
| 신호 전송률 (S) | 1초당 전송된 신호 요소 수 | 보오 (baud) |

$$S = \frac{N}{r}, \quad r = \frac{\text{데이터 요소}}{\text{신호 요소}}$$

---

### 문제점

- **기준선 표류 (Baseline Wandering)**
  - 수신측은 신호 평균값(기준선)으로 0과 1을 구분
  - 0 또는 1이 연속으로 길게 이어지면 기준선이 위아래로 밀려 이동
  - 이후 신호가 0인지 1인지 판단 불가
  - > 비유: 저울이 한쪽으로 계속 쏠리면 나중엔 무거운지 가벼운지 구분 못 하는 것

- **직류 성분 (DC Component)**
  - 0 또는 1이 길게 이어지면 신호가 거의 평평해짐 → 직류(DC)에 가까워짐
  - 저주파 성분은 전선·변압기가 물리적으로 통과시키지 못함 → 신호 왜곡/손실
  - > 비유: 스피커에 초저음을 계속 틀면 스피커가 감당 못 하고 망가지는 것처럼, 회선도 직류 성분을 처리 못 함

- **자기 동기화 (Self-Synchronization)**
  - 송신측이 비트를 보내는 타이밍과 수신측이 읽는 타이밍이 정확히 일치해야 함
  - 수신측 클럭이 조금이라도 빠르거나 느리면 비트를 잘못 읽음
  - 예) 10비트를 보냈는데 수신측이 9비트 또는 11비트로 읽어버리는 상황 발생
  - > 비유: 두 사람이 박자 맞춰 손뼉을 치는데 한 명이 조금씩 빨라지면 나중엔 완전히 어긋나는 것

---

### 부호화 방식

**극형 (Polar)** — 양/음 두 가지 전압 준위 사용, 직류 성분 완화

| 방식 | 설명 |
|---|---|
| NRZ (Non-Return-to-Zero) | 비트 간격 동안 전압 유지 |
| RZ (Return-to-Zero) | 비트 간격 중간에 0으로 복귀 |
| 맨체스터 | 비트 간격 중간에 반전 → 동기화 + 비트 표현 동시 해결 |
| 차분 맨체스터 | 중간 반전은 동기화용, 시작점 전이 여부로 비트 식별 (전이=0, 무변화=1) |

> - **맨체스터가 문제점을 해결하는 방법**:
>   - 비트 중간에 반드시 전이(transition)를 넣음
>   - 전이가 항상 있으므로 기준선 안정화 + 자기 동기화 동시 해결
>   - 직류 성분도 양/음이 균형을 이뤄 제거됨

---

**다준위 방식 (Multilevel)**

- **4D-PAM5**: 4개 회선 동시 전송, 5개 준위 (-2, -1, 0, 1, 2) → 1Gbps LAN에 사용

## 4.2 아날로그 → 디지털 변환 (PCM)

**펄스 코드 변조 (Pulse Code Modulation)** — 아날로그 신호를 디지털 데이터로 변환하는 표준 기법

---

### PCM 3단계 — **표양부** 로 외우기

**1단계: 표본화 (Sampling)**
- 일정 간격마다 신호의 진폭을 측정
- 나이퀴스트 정리: 표본화율 ≥ 최대 주파수 × 2

> 예) 전화 음성 (최대 4,000Hz) → 표본화율 = 8,000 samples/s

---

**2단계: 양자화 (Quantizing)**
- 표본화된 값을 L개 구간 중 하나의 정수값으로 근사
- 양자화 잡음 발생 (근사 과정에서 생기는 오차)

---

**3단계: 부호화 (Encoding)**
- 각 표본을 nb 비트 코드로 변환

$$\text{Bit rate} = f_s \times n_b$$

> 예) 음성 디지털화 (8bit/sample): 8000 × 8 = 64,000 bps

---

## 4.3 전송 방식

### 병렬 전송 vs 직렬 전송

| | 병렬 전송 | 직렬 전송 |
|---|---|---|
| 방식 | n개 비트를 동시에 전송 | 1비트씩 순차 전송 |
| 전선 수 | n개 | 1개 |
| 속도 | 직렬 대비 n배 빠름 | 상대적으로 느림 |
| 비용 | 높음 | 낮음 (1/n) |
| 거리 | 짧은 거리 | 장거리 가능 |

---

### 직렬 전송 방식

**비동기식 (Asynchronous)**
- 바이트 앞에 시작 비트(0), 끝에 정지 비트(1) 추가
- 총 8 데이터 비트 + 2 추가 비트
- 수신측이 각 바이트마다 재동기화
- 저속, 단거리에 적합

---

**동기식 (Synchronous)**
- 다수의 바이트를 프레임 단위로 전송, 바이트 간 간격 없음
- 비동기식보다 빠름, 고속 응용에 유리
- 동기화는 데이터 링크 계층에서 처리
