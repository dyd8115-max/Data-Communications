# Data Communications — 4. 디지털 전송

## 4.1 회선 부호화 (Line Coding)

2진 비트 데이터를 디지털 신호로 변환하는 작업

> - 비트(0,1)는 논리적 개념 → 물리적으로 전송 불가
> - 전압(+V, -V)으로 변환해야 실제 회선으로 전송 가능
> - 어떻게 변환하느냐에 따라 NRZ, RZ, 맨체스터 등 방식이 나뉨
> - 비유: 모스부호처럼 문자를 신호로 변환하는 것과 같음

### 기본 개념

| 용어 | 설명 | 단위 | 비고 |
|---|---|---|---|
| 데이터 요소 (n) | 데이터를 나타내는 가장 작은 단위 | 비트 (bit) | - |
| 신호 요소 | 디지털 신호의 가장 짧은 단위 | - | - |
| r | 신호 요소 1개당 비트 수 | - | r↑ → 한 신호에 더 많은 비트 |
| L | 신호 준위 수 | - | L = 2^r |
| 데이터 전송률 (N) | 1초당 전송된 비트 수 | bps | - |
| 신호 전송률 (S) | 1초당 전송된 신호 요소 수 | 보오 (baud) | - |

$$S = \frac{N}{r}, \quad r = \frac{\text{데이터 요소}}{\text{신호 요소}}$$

### 문제점
회선 부호화 시 **디지털 신호 자체의 특성**으로 인해 발생하는 3가지 문제

---

- **기준선 표류 (Baseline Wandering)**
  - 수신측은 신호 평균값(기준선)으로 0과 1을 구분
  - 0 또는 1이 연속으로 길게 이어지면 기준선이 위아래로 밀려 이동
  - 이후 신호가 0인지 1인지 판단 불가
  - > 비유: 저울이 한쪽으로 계속 쏠리면 나중엔 무거운지 가벼운지 구분 못 하는 것

---

- **직류 성분 (DC Component)**
  - 0 또는 1이 길게 이어지면 신호가 거의 평평해짐 → 직류(DC)에 가까워짐
  - 저주파 성분은 전선·변압기가 물리적으로 통과시키지 못함 → 신호 왜곡/손실
  - > 비유: 스피커에 초저음을 계속 틀면 스피커가 감당 못 하고 망가지는 것처럼, 회선도 직류 성분을 처리 못 함

---

- **자기 동기화 (Self-Synchronization)**
  - 송신측이 비트를 보내는 타이밍과 수신측이 읽는 타이밍이 정확히 일치해야 함
  - 수신측 클럭이 조금이라도 빠르거나 느리면 비트를 잘못 읽음
  - 예) 10비트를 보냈는데 수신측이 9비트 또는 11비트로 읽어버리는 상황 발생
  - > 비유: 두 사람이 박자 맞춰 손뼉을 치는데 한 명이 조금씩 빨라지면 나중엔 완전히 어긋나는 것

---

### 부호화 방식

**극형 (Polar)** — 양/음 두 가지 전압 준위 사용, 직류 성분 완화

- 1=+V, 0=-V 두 가지 전압으로 비트 표현
- 단극형(0V/+V)과 달리 양음이 균형을 이뤄 직류 성분 감소
- NRZ, RZ, 맨체스터, 차분 맨체스터 모두 극형 계열

---

**NRZ (Non-Return-to-Zero)**
- 비트 간격 동안 전압 유지, 0으로 복귀하지 않음

| 방식 | 설명 | 특징 |
|---|---|---|
| **NRZ-L (Level)** | 비트값 그대로 전압 유지 · 1=+V, 0=-V | 직관적, 구현 단순 |
| **NRZ-I (Invert)** | 1이 올 때마다 전압 반전 · 0이면 그대로 유지 | 연속 1에서 NRZ-L보다 동기화 유리 |

- 장점: 구현 단순, 대역폭 효율 좋음
- 단점: 기준선 표류, 자기 동기화 문제 (변화가 없으니 수신측이 몇 개인지 모름)
- 사용: USB, 시리얼 통신 (짧은 거리)

---

**RZ (Return-to-Zero)**
- 비트 중간에 무조건 0으로 복귀 (1: +V→0, 0: -V→0)
- 비트당 전압이 2번 움직임 → 대역폭 NRZ의 2배 필요
- 장점: 동기화 쉬움 (항상 0 복귀로 타이밍 잡힘)
- 단점: 대역폭 2배 소모
- 사용: 광섬유 통신 일부

> - **왜 대역폭 2배인가**: NRZ는 비트당 변화 최대 1번, RZ는 무조건 2번 → 신호가 2배 빠르게 변함 → 더 높은 주파수 필요 → 대역폭 2배

---

**맨체스터 (Manchester)**
- 비트 중간에 무조건 반전. 반전 방향이 비트를 나타냄
  - 1 = 중간에 LOW→HIGH (↑)
  - 0 = 중간에 HIGH→LOW (↓)
- 절대적 기준 → 신호 극성 뒤집히면 방향 기준이 바뀌어 오독 가능
- 장점: 동기화 + 직류 성분 없음, 구현 단순
- 단점: 대역폭 NRZ의 2배
- 사용: 이더넷 (10BASE-T), RFID

---

**차분 맨체스터 (Differential Manchester)**
- 중간 반전은 항상 있음 (동기화용)
- 비트 구분은 시작점 수직선(전이) 유무로
  - 0 = 비트 시작점에 전이 있음 (수직선 2개)
  - 1 = 비트 시작점에 전이 없음 (수직선 1개, 중간만)
- 상대적 기준 → 신호 극성 뒤집혀도 전이 유무는 안 바뀜 → 잡음에 강함
- 장점: 잡음/극성 반전에 강함, 동기화 좋음
- 단점: 대역폭 NRZ의 2배, 구현 복잡
- 사용: 토큰링, 자기테이프

> - **맨체스터 vs 차분 맨체스터 핵심 차이**:
>   - 맨체스터 = 나침반 (방향 자체가 의미) → 신호 뒤집히면 망함
>   - 차분 맨체스터 = 변화 유무만 봄 → 신호 뒤집혀도 상관없음

---

| | NRZ | RZ | 맨체스터 | 차분 맨체스터 |
|---|---|---|---|---|
| 대역폭 | 1배 | 2배 | 2배 | 2배 |
| 동기화 | 약함 | 좋음 | 좋음 | 좋음 |
| 잡음 저항 | 보통 | 보통 | 보통 | 강함 |
| 구현 | 단순 | 단순 | 단순 | 복잡 |
| 사용 | USB | 광섬유 | 이더넷 | 토큰링 |

---

**다준위 방식 (Multilevel)**

- **4D-PAM5**: 4개 회선 동시 전송, 5개 준위 (-2, -1, 0, 1, 2) → 1Gbps LAN에 사용

> - **왜 다준위를 쓰는가**:
>   - 2준위(+V/-V)는 신호 1개당 1비트만 표현 가능
>   - 준위를 5개로 늘리면 신호 1개당 log₂5 ≈ 2.32비트 표현 가능
>   - 4개 회선 동시 사용 → 4 × 2.32 ≈ 9비트를 한 번에 전송
>   - 대역폭은 그대로인데 전송 효율이 올라감 → 1Gbps 달성 가능

**펄스 코드 변조 (Pulse Code Modulation)** — 아날로그 신호를 디지털 데이터로 변환하는 표준 기법

### PCM 3단계

**1단계: 표본화 (Sampling)**
- 일정 간격마다 신호의 진폭을 측정
- 나이퀴스트 정리: 표본화율 ≥ 최대 주파수 × 2

> 예) 전화 음성 (최대 4,000Hz) → 표본화율 = 8,000 samples/s

**2단계: 양자화 (Quantizing)**
- 표본화된 값을 L개 구간 중 하나의 정수값으로 근사
- 양자화 잡음 발생 (근사 과정에서 생기는 오차)

$$\Delta = \frac{V_{max} - V_{min}}{L}$$

- Δ (delta): 양자화 간격 (구간 크기)
- V_max (maximum voltage): 신호의 최대 전압값
- V_min (minimum voltage): 신호의 최소 전압값
- L (levels): 양자화 준위 수
- L↑ → Δ↓ → 더 세밀하게 근사 → 양자화 잡음 감소

**3단계: 부호화 (Encoding)**
- 각 표본을 nb 비트 코드로 변환

$$\text{Bit rate} = f_s \times n_b$$

- Bit rate: 초당 전송 비트 수 (bps)
- f_s (sampling frequency): 1초당 표본화 횟수 (samples/s)
- n_b (number of bits per sample): 표본당 비트 수

> 예) 음성 디지털화 (8bit/sample): 8000 × 8 = 64,000 bps

> - **PCM과 광대역의 관계**:
>   - PCM = 아날로그 → 디지털 (저장/처리 목적)
>   - 광대역 = 디지털 → 아날로그 변조 후 전송 (전송 목적)
>   - 실제로는 둘이 이어져서 쓰임: 아날로그 음성 → [PCM] → 디지털 → [광대역 변조] → 전송
>   - 예) 인터넷 전화(VoIP): 음성을 PCM으로 디지털화 → 광대역으로 인터넷에 실어 전송

## 4.3 전송 방식

### 병렬 전송 vs 직렬 전송

| | 병렬 전송 | 직렬 전송 |
|---|---|---|
| 방식 | n개 비트를 동시에 전송 | 1비트씩 순차 전송 |
| 전선 수 | n개 | 1개 |
| 속도 | 직렬 대비 n배 빠름 | 상대적으로 느림 |
| 비용 | 높음 | 낮음 (1/n) |
| 거리 | 짧은 거리 | 장거리 가능 |

### 직렬 전송 방식

**비동기식 (Asynchronous)**
- 바이트 앞에 시작 비트(0), 끝에 정지 비트(1) 추가
- 총 8 데이터 비트 + 2 추가 비트
- 수신측이 각 바이트마다 재동기화

**동기식 (Synchronous)**
- 다수의 바이트를 프레임 단위로 전송, 바이트 간 간격 없음
- 비동기식보다 빠름, 고속 응용에 유리
- 동기화는 데이터 링크 계층에서 처리
