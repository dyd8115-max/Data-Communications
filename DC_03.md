# Data Communications — 3. 물리층 개요 (1)
## 3.1 데이터와 신호
| 구분 | 데이터 | 신호 | 장점 | 단점 |
|---|---|---|---|---|
| 아날로그 | 음성, 온도, 시간 등 연속적인 값 | 연속적인 파형 | 자연 신호 그대로 표현, 변환 손실 없음 | 잡음에 취약, 복사 시 품질 저하, 암호화 어려움 |
| 디지털 | 0과 1로 저장된 데이터 | 이산적, 제한된 값 (0 또는 1) | 잡음에 강함, 복사/저장 시 품질 유지, 암호화 용이 | 변환 시 정보 손실 가능 (샘플링), 대역폭 많이 필요 |

**변환 장치**
- **모뎀**: 디지털 데이터 ↔ 아날로그 신호
- **코덱**: 아날로그 데이터 ↔ 디지털 신호

## 3.2 주기 아날로그 신호
### 정현파 (Sine Wave)
아날로그 주기 신호의 가장 기본적인 형태
$$s = A \sin(2\pi ft + \phi)$$
| 요소 | 기호 | 설명 |
|---|---|---|
| 순간 진폭 | $s$ | 특정 시점의 신호 크기 |
| 최대 진폭 | $A$ | 신호의 최대 세기 |
| 주파수 | $f$ | 1초당 반복 횟수 (Hz) - 높을수록 파형 촘촘 → 많은 데이터 전송 / 낮을수록 파형 느슨 → 적은 데이터 전송 |
| 주기 | $T$ | 신호가 한 번 반복되는 데 걸리는 시간 (초), $T = \frac{1}{f}$ |
| 위상 | $\phi$ | 파형의 시작 위치 |
| **시간** | $t$ | **현재 시각 (초)** — 흐르는 시간, 주기 $T$와 다름 |

### sin 값 참조표
> 공식의 $\sin(2\pi ft + \phi)$ 부분 — $f = 1$ Hz($T = 1$초), $\phi = 0$ 기준

| 각도 (°) | 각도 (rad) | **t (초)** | sin 값 | 파형 상태 |
|:---:|:---:|:---:|:---:|---|
| 0° | 0 | **t = 0** | 0 | 시작 (파형 출발) |
| 30° | π/6 | t = 1/12 | 0.5 | 올라가는 중 |
| 45° | π/4 | t = 1/8 | ≈ 0.707 | 올라가는 중 |
| 60° | π/3 | t = 1/6 | ≈ 0.866 | 거의 정점 |
| **90°** | **π/2** | **t = 1/4** | **1 (최대)** | **정점 → 진폭 A** |
| 180° | π | **t = 1/2** | 0 | 절반 주기 완료 |
| **270°** | **3π/2** | **t = 3/4** | **-1 (최솟값)** | **골짜기 → -A** |
| 360° | 2π | **t = 1 = T** | 0 | 한 주기 완료 → $t = T$ |

> - sin 범위는 항상 **-1 ~ 1**
> - 공식에서 $A$는 sin 값에 곱해지는 스케일 → 실제 진폭 범위는 **-A ~ A**
> - $\phi$ (위상)는 파형의 시작 각도 → $\phi = 90°$이면 처음부터 최댓값 A에서 출발

### 파장 (Wavelength)
단순 신호가 **한 주기 동안 진행하는 거리**
$$\lambda = \frac{c}{f} = c \cdot T$$

$c$: 빛의 속도 ($3 \times 10^8$ m/s)

$f$: 주파수

$T$: 주기
> 예) 빨간색 빛 ($f = 4 \times 10^{14}$ Hz) → $\lambda = \frac{3 \times 10^8}{4 \times 10^{14}} = 7.5 \times 10^{-7}$ m

### 복합 신호
실제 데이터 통신에 사용되는 신호는 복합 신호

- **주기적 복합 신호**
  - 일정한 패턴이 반복되는 복합 신호
  - 여러 정현파의 합으로 분해 가능 (푸리에 급수)
  - 이산적인 주파수 성분 (딱 떨어지는 값)
  - ex: 악기 소리, 전력선 신호

- **비주기적 복합 신호**
  - 패턴이 반복되지 않는 복합 신호
  - 연속적인 주파수 범위를 가짐 (푸리에 변환)
  - 데이터 통신에서 대부분 해당
  - ex: 음성, 데이터 신호

| | 주기적 | 비주기적 |
|---|---|---|
| **반복** | 있음 | 없음 |
| **주파수 성분** | 이산적 (딱 떨어짐) | 연속적 (범위) |
| **분석 방법** | 푸리에 급수 | 푸리에 변환 |
| **예시** | 악기 소리, 전력선 | 음성, 데이터 신호 |

> - **푸리에 급수 (Fourier Series)**
>   - 주기적 복합 신호를 여러 개의 단순 정현파 합으로 분해하는 수학적 방법
>   - 각 성분의 주파수는 기본 주파수의 정수배
> - **푸리에 변환 (Fourier Transform)**
>   - 비주기적 복합 신호를 연속적인 주파수 성분으로 분석하는 방법
>   - 신호가 어떤 주파수를 얼마나 포함하는지 파악 가능 → 대역폭 계산에 활용

### 대역폭 (Bandwidth)
$$\text{대역폭} = f_{\max} - f_{\min}$$
복합 신호에 포함된 **최고 주파수 - 최저 주파수**

## 3.3 디지털 신호
- 디지털 신호 = 무한대의 주파수를 갖는 복합 아날로그 신호
- 대역폭 = 무한대

> - **왜 무한대인가**:
>   - 디지털 신호의 수직 전환(0↔1)을 수학적으로 표현하려면 무한히 많은 정현파를 합쳐야 함
>   - 고주파 성분이 많을수록 모서리가 날카로워짐
>   - 완벽한 수직 전환 = 무한대의 고주파 필요
> - 실제 전송에서는 대역폭이 제한되므로 모서리가 약간 뭉개진 형태로 근사 전송

### 전송 방식
| 방식 | 설명 | 특징 |
|---|---|---|
| 기저대역 (Baseband) | 디지털 신호를 그대로 채널로 전송 | 저대역 통과 채널 사용, LAN에 주로 사용 |
| 광대역 (Wideband) | 디지털 신호를 아날로그로 변조 후 전송 | 밴드패스 채널 사용, 변복조 필요 |

**기저대역 전송**
- 저대역 통과 채널(low-pass channel) 사용 — 주파수 0부터 시작
- 아날로그 변환 없이 그대로 전송
- 동축케이블, 광섬유, LAN에서 사용
- 대역폭이 넓을수록 더 많은 주파수 성분 전달 가능 → 디지털 신호를 더 정확하게 표현 (모서리가 날카로워짐)

**광대역 전송**
- 디지털 신호 → 아날로그 신호로 변조(modulation)
- 밴드패스(bandpass) 채널 사용
- 여러 신호를 동시에 다른 주파수 대역에 실어 전송 가능 (주파수 분할)
- 장거리 전송에 유리
- ex: 케이블 TV, DSL, 와이파이, LTE
