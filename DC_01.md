# Data Communications — 1. 데이터 통신의 개요
## 1.1 데이터 통신
- **전기통신 (Telecommunication)**: 먼 거리에서 행해지는 통신 (tele = 멀다)
- **데이터 (Data)**: 송수신자 간 합의된 형태로 표현된 정보 (0과 1)
- **데이터 통신**: 통신매체를 통해 두 장치 간 데이터를 교환하는 것
### 구성 요소
| 요소 | 설명 | 예시 |
|---|---|---|
| 송신자 (Sender) | 메시지를 보내는 장치 | 컴퓨터, 전화기, 카메라 |
| 수신자 (Receiver) | 메시지를 받는 장치 | 컴퓨터, 전화기, TV |
| 메시지 (Message) | 전송되는 정보 | 문자, 숫자, 소리, 영상 |
| 전송매체 (Medium) | 메시지를 전달하는 물리적 경로 | 꼬임선, 동축선, 광케이블, 무선 |
| 프로토콜 (Protocol) | 데이터 통신 수행 규칙들의 집합 (상호 합의) | - |
### 데이터 표현
- **문자**: 비트 패턴(0과 1)으로 표현
  - ASCII: 7bit / 확장 ASCII: 8bit / 유니코드: 32bit
- **수**: 비트 패턴으로 표현
- **화상**: 픽셀(pixel) 단위, 흑백(1~2bit) / 컬러(RGB, CMY)
- **오디오**: 연속 신호를 샘플링
- **비디오**: 연속적인 화상의 조합
### 데이터 흐름
| 방식 | 설명 | 예시 |
|---|---|---|
| 단방향 (Simplex) | 한쪽 방향으로만 통신 | 키보드, 모니터 |
| 반이중 (Half-duplex) | 송수신 가능하나 동시 불가 | CB 라디오, 무전기 |
| 전이중 (Full-duplex) | 동시 양방향 송수신 가능 | 전화 |

## 1.2 네트워크
전송 매체 링크로 연결된 장치(노드)들의 모임
### 네트워크 평가 기준
- **성능**: 처리율(throughput), 지연(delay)
- **신뢰성**: 고장 빈도, 복구 시간
- **보안**: 불법 접속 및 바이러스로부터 보호

> - **우회 가능성**: 링크나 장치 고장 시 대체 경로로 우회 가능한 구조일수록 신뢰성 높음 (예: 그물형 토폴로지)

### 연결 유형
- **점대점 (Point-to-Point)**: 두 기기 간 전용 채널 (ex: 블루투스 1:1 연결, 회사-데이터센터 전용선)
- **다중점 (Multipoint)**: 3개 이상 기기가 하나의 링크 공유 (공간적/시간적 공유) (ex: 와이파이, 강의실 LAN)

> - **보안**: 점대점 강함 (전용 채널) / 다중점 약함 (링크 공유)
> - **비용**: 점대점 비쌈 (1:1마다 회선 필요) / 다중점 저렴 (링크 공유)
### 접속 형태 (Topology)
| 종류 | 특징 | 데이터 흐름 | 장점 | 단점 | 사용 분야 |
|---|---|---|---|---|---|
| 그물형 (Mesh) | 모든 장치가 점대점 연결, n(n-1)/2개 링크 필요 | 점대점 · 전이중 | 높은 안정성, 결함 분리 용이, 고장 시 우회 가능, 중간 경유 없이 직접 전달로 보안 강함 | 케이블·포트 비용 큼 | 군사 네트워크, 금융 백본망 |
| 스타형 (Star) | 중앙 허브에 모두 연결, **가장 많이 사용** | 점대점 · 전이중 | 비용 적음, 설치 쉬움 | 허브 고장 시 전체 마비, 허브 경유 필수로 보안 취약 | 가정/회사 LAN, 와이파이 |
| 버스형 (Bus) | 다중점, 탭과 유도선으로 연결 | 다중점 · 반이중 | 설치 쉬움, 케이블 최소 | 중추 케이블 결함 시 전체 영향 | 초기 이더넷 (현재 거의 미사용) |
| 링형 (Ring) | 양쪽 장치와 점대점 연결, 각 장치에 중계기 포함 | 점대점 · 단방향 순환 | 설치·재구성 쉬움 | 단방향 시 링 결함 → 전체 마비 | 광통신 백본 FDDI (현재 거의 미사용) |
| 혼합형 (Hybrid) | 위 토폴로지 조합 | 조합에 따라 다름 | - | - | 대규모 기업망 |

> - **토큰 (Token)**: 링을 순환하는 전송 권한 신호. 토큰을 가진 장치만 데이터 전송 가능 → 충돌 방지 → **반이중**
> - **듀얼 링 (Dual Ring)**: 링을 2개 구성 (시계/반시계 방향). 주 링 고장 시 백업 링으로 자동 우회 → 안정성 확보 → **전이중**

| 항목 | 1위 | 2위 | 3위 | 4위 |
|---|---|---|---|---|
| **가격 (저렴)** | 버스 | 링 | 스타 | 그물 |
| **보안** | 그물 | 스타 | 링 | 버스 |
| **안정성** | 그물 | 스타 | 링 | 버스 |
| **설치 난이도 (쉬움)** | 버스 | 스타 | 링 | 그물 |
| **확장성** | 스타 | 버스 | 링 | 그물 |

## 1.3 네트워크 유형
| 종류 | 범위 | 특징 |
|---|---|---|
| LAN | 단일 건물/학교 | 자원 공유 목적, 버스·링·스타형 사용 |
| WAN | 국가/대륙/전세계 | 거리 제한 없음, 통신회사가 임대 운영 |
> - **internet (소문자)**: 두 개 이상의 네트워크가 서로 연결된 것 (일반 개념)
> - **Internet (대문자)**: 전 세계를 연결하는 특정 글로벌 네트워크 (우리가 사용하는 그 인터넷)
### 교환 방식
| 항목 | 회선 교환 (Circuit-switched) | 패킷 교환 (Packet-switched) |
|---|---|---|
| **개념** | 통신 전 전용 경로 확보, 종료까지 독점 사용 | 데이터를 패킷으로 쪼개 각각 독립 전송, 경로 달라도 됨 |
| **전송 단위** | 회선 (연속 스트림) | 패킷 (데이터 조각) |
| **연결 방식** | 통신 전 전용 회선 설정 필요 | 연결 설정 없이 바로 전송 |
| **지연** | 일정 (전용 회선) | 가변 (혼잡도에 따라 다름) |
| **비용** | 고정 (사용량 무관) | 효율적 (사용한 만큼) |
| **효율** | 낮음 (데이터 없어도 회선 점유 → 낭비) | 높음 (데이터 있을 때만 사용, 나머지는 공유) |
| **용량** | 회선 대역폭으로 고정 | 네트워크 상황에 따라 유동적 |
| **예시** | 전화망, 전용선 | 인터넷, 이메일, 카카오톡 |

> - **비유**: 회선 교환 = 트럭 전세 (비어있어도 내 것) / 패킷 교환 = 합송 택배 (여러 사람 짐을 같은 트럭에)
> - **현재 주류**: 패킷 교환. 인터넷 대부분이 패킷 교환 기반
> - **회선 교환**: 품질 보장이 필요한 음성통화에 여전히 사용 (단, VoIP 보급으로 점차 감소)
---
### 인터넷 구조
> - **백본**: Sprint, Verizon, AT&T 등 통신사 소유, 대등점(peering point)으로 연결
> - **제공자 네트워크**: 요금 지불하고 백본 이용 (ISP)
> - **소비자 네트워크**: 인터넷 말단, 서비스 이용
