## CH01 · 데이터 통신 기본

### 구성 5요소
| 요소 | 설명 |
|---|---|
| 송신자 | 메시지를 보내는 장치 |
| 수신자 | 메시지를 받는 장치 |
| 메시지 | 전송되는 정보 |
| 전송매체 | 물리적 경로 (케이블, 무선 등) |
| **프로토콜** | **데이터 통신 수행 규칙 집합 ← 자주 틀림** |

### 데이터 흐름
| 방식 | 설명 | 예시 |
|---|---|---|
| 단방향 (Simplex) | 한쪽 방향만 | 키보드, 모니터 |
| 반이중 (Half-duplex) | 양방향, 동시 불가 | 무전기, CB라디오 |
| 전이중 (Full-duplex) | 동시 양방향 | 전화 |

### 교환 방식
| | 회선 교환 | 패킷 교환 |
|---|---|---|
| 개념 | 전용 경로 미리 확보, 독점 사용 | 패킷으로 쪼개 독립 전송 |
| 효율 | 낮음 (데이터 없어도 점유) | 높음 (있을 때만 사용) |
| 예시 | 전화망 | 인터넷, 카카오톡 |
| 비유 | 트럭 전세 | 합송 택배 |

> 현재 주류 = **패킷 교환**

## CH01 · 토폴로지

| 종류 | 특징 | 약점 |
|---|---|---|
| 그물 (Mesh) | n(n-1)/2 링크, 가장 안전 | 비용 큼 |
| **스타 (Star)** | **가장 많이 사용**, LAN | 허브 고장 → 전체 마비 |
| 버스 (Bus) | 설치 쉬움, 케이블 최소 | 중추 결함 → 전체 영향 |
| 링 (Ring) | 각 장치에 중계기 | 링 결함 → 전체 마비 |

**순위 정리**
- 가격 (저렴): 버스 > 링 > 스타 > 그물
- 보안/안정성: 그물 > 스타 > 링 > 버스
- 설치 쉬움: 버스 > 스타 > 링 > 그물

## CH02 · OSI 7계층 vs TCP/IP

| # | OSI | TCP/IP | 전달 단위 | 주소 |
|---|---|---|---|---|
| 7·6·5 | 응용·표현·세션 | 응용층 | - | 이름(도메인) |
| 4 | 전송층 | 전송층 | 세그먼트 | 포트번호 |
| 3 | 네트워크층 | 인터넷층 | 패킷 | IP주소 |
| 2 | 데이터링크층 | 데이터링크층 | 프레임 | MAC주소 |
| 1 | 물리층 | 물리층 | 비트 | 없음 |

> OSI = 교육용 개념 / TCP/IP = 실제 사용
> TCP/IP는 세션·표현층이 응용층으로 합쳐져 5계층

### MAC vs IP
| | MAC | IP |
|---|---|---|
| 계층 | 2계층 | 3계층 |
| 범위 | LAN 안에서만 유효 | 전체 경로 유지 |
| 변경 | 홉마다 교체 | 출발~목적지 유지 |
| 비유 | 주민등록번호 (정거장) | 집 주소 (최종 목적지) |

### Switch vs Router
| | Switch | Router |
|---|---|---|
| 주소 | MAC | IP |
| 역할 | LAN 내 장치 연결 | 네트워크 간 연결 |
| 계층 | 2계층 | 3계층 |

## CH03 · 신호 기초

### 정현파

$$s = A\sin(2\pi ft + \phi)$$

| 요소 | 의미 |
|---|---|
| A | 최대 진폭 |
| f | 주파수 (Hz) — **높을수록 데이터 전송↑** |
| T | 주기 = 1/f |
| φ | 위상 (시작 위치) |

- 파장: `λ = c/f` (c = 빛의 속도 3×10⁸ m/s)
- 대역폭: `BW = f_max - f_min`

### 아날로그 vs 디지털
| | 아날로그 | 디지털 |
|---|---|---|
| 장점 | 자연 신호 그대로, 변환 손실 없음 | 잡음에 강함, 암호화 용이, 복사 품질 유지 |
| 단점 | 잡음 취약, 복사 품질 저하 | **변환 시 정보 손실, 대역폭 많이 필요** |

> 디지털 신호 = 이론상 무한 대역폭 (수직 전환에 무한 고주파 필요)

### 전압 준위
| | 단극형 (Unipolar) | 극형 (Polar) |
|---|---|---|
| 준위 | +V / 0V | +V / -V |
| 직류 성분 | 심함 | 적음 (양음 균형) |

## CH03 · 전송률 한계 공식

### 나이퀴스트 (잡음 없는 채널)
```
Bit Rate = 2 × BW × log₂L
```
- L = 신호 준위 수

### 섀논 (잡음 있는 채널)
```
C = BW × log₂(1 + SNR)
```
- 이론적 상한값
- 1을 더하는 이유: SNR=0일 때 log₂(0) = 무한대 방지 → C=0으로 자연스럽게 처리

### 관련 공식
```
SNR_dB = 10 × log₁₀(SNR)
감쇠 dB = 10 × log₁₀(P₂/P₁)
```

> 실제 설계: 섀논으로 상한 구하고 → 나이퀴스트로 신호 준위 수 계산

## CH04 · 회선 부호화 기본 개념

$$S = \frac{N}{r}$$

- S: 신호 전송률 (baud) — 1초당 신호 요소 수
- N: 데이터 전송률 (bps) — 1초당 비트 수
- r: 신호 요소당 데이터 요소 수

### 회선 부호화 문제점
| 문제 | 원인 | 해결 |
|---|---|---|
| 기준선 표류 | 0 또는 1이 연속 → 기준선 이동 | 맨체스터 |
| 직류 성분 | 저주파 성분 → 전선/변압기 통과 불가 | 극형 방식 |
| 자기 동기화 | 클럭 불일치 → 비트 잘못 읽음 | 맨체스터 |

---

## CH04 · 회선 부호화 방식

| 방식 | 비트 표현 | 대역폭 | 동기화 | 잡음 저항 | 사용 |
|---|---|---|---|---|---|
| NRZ | 전압 유지 (1=+V, 0=-V) | 1배 | 약함 | 보통 | USB, 시리얼 |
| RZ | 중간에 0 복귀 | 2배 | 좋음 | 보통 | 광섬유 일부 |
| 맨체스터 | 중간 반전 방향 (↑=1, ↓=0) | 2배 | 좋음 | 보통 | 이더넷, RFID |
| 차분 맨체스터 | 시작점 전이 유무 (전이=0, 무=1) | 2배 | 좋음 | **강함** | 토큰링 |

> **맨체스터 vs 차분 맨체스터 핵심 차이**
> - 맨체스터 = 반전 방향이 비트 의미 → 신호 극성 뒤집히면 오독
> - 차분 맨체스터 = 전이 유무가 비트 의미 → 신호 극성 뒤집혀도 전이 유무는 안 바뀜 → 잡음에 강함

### 다준위 방식
- **4D-PAM5**: 선 4개 × 5준위(-2,-1,0,+1,+2) → 선당 log₂5 ≈ 2.32비트 → 1Gbps LAN

## CH04 · PCM (아날로그 → 디지털 변환)

### 3단계: **표양부** 로 외우기

| 단계 | 이름 | 내용 |
|---|---|---|
| 1 | **표**본화 (Sampling) | 일정 간격으로 진폭 측정. 표본화율 ≥ 최대주파수 × 2 |
| 2 | **양**자화 (Quantizing) | 측정값을 정수로 근사 → 양자화 잡음 발생 |
| 3 | **부**호화 (Encoding) | n비트 코드로 변환 |

### PCM 공식
```
양자화 간격: Δ = (Vmax - Vmin) / L
비트율:      Bit rate = fs × nb
```
- Δ: 구간 크기. L 클수록 Δ 작아짐 → 잡음 감소
- Vmax / Vmin: 신호 최대/최소 전압
- L: 양자화 준위 수
- fs: 1초당 표본화 횟수
- nb: 표본당 비트 수

> 예) 음성: 8,000 samples/s × 8bit = **64,000 bps**

## CH04 · 전송 방식

### 병렬 vs 직렬
| | 병렬 | 직렬 |
|---|---|---|
| 방식 | n비트 동시 전송 | 1비트씩 순차 |
| 속도 | n배 빠름 | 상대적으로 느림 |
| 비용 | 높음 | 낮음 |
| 거리 | 단거리 | 장거리 |

**직렬 전송 종류**
- **비동기식**: 시작 비트(0) + 데이터 + 정지 비트(1) — 바이트마다 재동기화
- **동기식**: 프레임 단위 전송, 간격 없음 — 고속 응용에 유리

## CH05 · 아날로그 전송 (변복조)

### 변조 방식
| 방식 | 변조 요소 | 특징 |
|---|---|---|
| ASK | 진폭 | 진폭만 변화, 주파수/위상 유지 |
| FSK | 주파수 | 주파수만 변화 |
| PSK | 위상 | 위상만 변화. QPSK = 4위상 → 신호당 2비트 |
| QAM | 진폭 + 위상 | ASK+PSK 조합. 고속 전송에 유리 |

### 신호율 공식

$$S = \frac{N}{r}$$

- S: 신호율 (baud)
- N: 비트율 (bps)
- r: 신호 요소당 비트 수

### 성운도 (Constellation Diagram)
- X축 = 동위상, Y축 = 구상
- 점 개수 많을수록 → 신호당 비트 ↑ → 효율 ↑ → 잡음에 취약

| 방식 | 점 수 | 비트/신호 |
|---|---|---|
| BPSK | 2 | 1 |
| QPSK | 4 | 2 |
| 16-QAM | 16 | 4 |
| 64-QAM | 64 | 6 |
